## 💡 技巧

**1. NVIDIA NeMo AutoModel：MoE 微调一行加速 3.4-3.7 倍**
基于 Transformers v5 加 Expert Parallelism、DeepEP 融合 all-to-all、TransformerEngine 内核；只需改一行 import，GPU 内存减 29-32%；550B A55B 在 16 节点 128 H100 上可训。
- [Hugging Face Blog](https://huggingface.co/blog/nvidia/accelerating-fine-tuning-nvidia-nemo-automodel)

**2. OpenRouter 零数据留存（ZDR）：97 款新模型，流量占比近半**
ZDR 在三层执行：账户级、护栏级（按 API Key / 组织成员）、单次请求级（传参数）；多供应商架构可借此避免供应商锁定同时控制数据留存。
- [OpenRouter 博客](https://openrouter.ai/blog/insights/when-zero-means-zero)

**3. 字节 AI Coding 实践：900 次实验显示主流模型代码正确率超 80%，可交付性仅 40-60 分**
TRAE 团队 AI 代码贡献率超 90%，人均需求吞吐率仅提升 60%；结合 Harness 基建后可交付性提至 80 分。提示：别只盯代码占比，要建可交付性度量 + Harness 治理。
- [火山引擎公众号](https://mp.weixin.qq.com/s/mdmaAyUIvxE8WT_GEbF2wQ)

**4. OLMo Hybrid vs Transformer：混合模型在实义词上优势明显，重复短语上几无优势**
7B 参数下，OLMo Hybrid 对名词/动词/形容词 loss gap 约 0.04，功能词约 0.02；但在重复 n-gram 和闭合括号上优势消失。提示：选模型要看具体任务分布。
- [Hugging Face Blog](https://huggingface.co/blog/allenai/hybrid-token-prediction)

**5. 字节豆包"专业版"上线：定位企业级 AI 办公**
