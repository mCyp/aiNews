## 💡 技巧

1. **构建 AI Agent：先设计路由（router），再选模型**
   路由决定请求去哪层模型。正确路由让 70-80% 流量跑在免费本地模型或异步推理上，AI 开销可降 90%+。Coinbase 把 token 用量翻倍时 AI 支出减半，依赖的就是更好的默认设置、路由与缓存。建议把 Agent 抽象拆成"技能分类器 → 路由器 → 模型选择器"三层。
   - [Tomer Tunguz](https://www.tomtunguz.com/ai-execution-routing)

2. **写可预测 AI Skill 的五条纪律**
   mattpocockuk 的 /writing-great-skills：① 用 description 当触发器，明确 model-invoked vs user-invoked；② 三层信息结构（主步骤 / 参考 / 外部文件）做渐进式披露；③ 每步写"完成标准"；④ 拆 Skill 是为了控制模型注意力；⑤ 用 leading word 压缩行为要求。五种失败模式：Premature completion / Duplication / Sediment / Sprawl / No-op，可以用 No-op 测试句子有效性。
   - [GitHub](https://github.com/mattpocock/skills/tree/main/skills/productivity/writing-great-skills)

3. **Meta 大规模 AI 存储蓝图的关键经验**
   训练栈迁到 BLOB 存储 + 闪存 pMax 延迟，避免单 GPU 慢速拖慢整批任务；统一数据湖让跨数据中心 GPU 高速数据注入；元数据查询从数百毫秒压到预测性低位延迟。
   - [Meta Engineering](https://engineering.fb.com/2026/07/01/data-infrastructure/metas-ai-storage-blueprint-at-scale)
